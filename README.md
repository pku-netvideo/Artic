## ZocoStream Video Encoding Pipeline

ZocoStream gets feedback region's bounding-box from the model, converts it into Kvazaar ROI maps, and produces ROI-encoded videos. Dataset paths and output paths are passed from the command line.

Run the following commands from the repository root. The examples below call scripts under `ZocoStream/` directly.

### File Structure
The downloaded StreamingBench video folder is referred to as `video_root` in the commands below. It should contain:

```text
<video_root>/
`-- sample_<sample_id>/
    `-- video.mp4
```

The other two root directories are user-defined output/work folders:

- `roi_root`: generated by this pipeline. It stores sampled 2 FPS frames and predicted ROI JSON files.
- `baseline_root`: generated by the encoding preparation and encoding stages. It stores source frame folders, Kvazaar ROI maps, and encoded ROI videos.

Expected layout after running the pipeline:

```text
<roi_root>/
`-- sample_<sample_id>_start_<start_time>.0s/
    |-- 00001.png
    |-- 00002.png
    `-- q_<original_index>/
        `-- pred_00001.json

<baseline_root>/
`-- sample_<sample_id>/
    |-- video_<start_time>s_png_<encode_fps>fps/
    |   `-- 000001.png
    |-- roi_<fps_tag>_q_<original_index>.txt
    `-- video_<bitrate_tag>_<start_time>s_q_<original_index>_roi.mp4
```

`get_frames.py` creates the source frame folder under `baseline_root`. `batch_process_robust.py` expects this folder to exist before encoding:

```text
<baseline_root>/sample_<sample_id>/video_<start_time>s_png_<encode_fps>fps/
```

### Prepare CSV

Use the DeViBench CSV and add a stable ID column before any filtering, splitting, or merging:

```python
import pandas as pd

df = pd.read_csv("datasets.csv")
df.insert(0, "original_index", range(len(df)))
df.to_csv("datasets_with_original_index.csv", index=False)
```

`original_index` is the zero-based row index of the original DeViBench CSV:

```text
original_index = row_number - 1
```

Keep `original_index` unchanged afterwards. It is used to align `q_<original_index>`, ROI maps, and encoded ROI videos.

Required columns:

| Stage | Required columns |
| --- | --- |
| ROI frame sampling | `sample_id`, `start_time` |
| ROI prediction | `sample_id`, `question`, `start_time` |
| Encoding frame extraction | `sample_id`, `start_time` |
| ROI encoding | `original_index`, `sample_id`, `start_time` |

### Frame Rate

This pipeline uses two frame rates:

- ROI prediction frames are sampled at 2 FPS by default with `sampling.py --target-fps 2`.
- ROI encoding uses 30 FPS by default with `get_frames.py --fps 30` and `batch_process_robust.py --encode_fps 30`.

You can adjust the encoding FPS for different encoding settings. If you change `--encode_fps`, keep the following values consistent:

1. The source frame extraction FPS, e.g. `get_frames.py --fps 30`.
2. The encoding FPS, e.g. `batch_process_robust.py --encode_fps 30`.
3. The shared source frame folder suffix under `baseline_root`, e.g. `video_3s_png_30fps` for 30 FPS.
4. The encoded frame count, computed as `encode_fps * target_seconds`. The duration is controlled by `get_frames.py --duration` and `batch_process_robust.py --target_seconds`.

### Run Pipeline

### 1. Sample frames at 2 FPS

```bash
python ZocoStream/sampling.py \
  --csv /path/to/datasets_with_original_index.csv \
  --base-video-dir /path/to/video_root \
  --output-dir /path/to/roi_root \
  --target-fps 2
```

This creates:

```text
<roi_root>/sample_<sample_id>_start_<start_time>.0s/00001.png
```

### 2. Predict ROI bounding boxes
Set the Ark API key before ROI prediction:

```bash
export ARK_API_KEY=your_key_here
```

You can also pass it directly with `--api-key`.

```bash
python ZocoStream/generate_roi.py \
  --csv /path/to/datasets_with_original_index.csv \
  --roi-root /path/to/roi_root \
  --frames-expected 16 \
  --history 3 \
  --pred 3 \
  --a-start 2 \
  --concurrency 16 \
  --model doubao-seed-1-8-251228
```

This creates:

```text
<roi_root>/sample_<sample_id>_start_<start_time>.0s/q_<row_index>/pred_00001.json
```

To make this folder match `q_<original_index>`, run ROI prediction on the original ordered CSV where `row_index == original_index`.

### 3. Extract shared source frames for encoding

```bash
python ZocoStream/get_frames.py \
  --csv /path/to/datasets_with_original_index.csv \
  --video_root /path/to/video_root \
  --baseline_root /path/to/baseline_root \
  --fps 30 \
  --duration 5 \
  --start_number 1
```

This only needs to be done once for each sample/start-time pair. All bitrate encodes share the same extracted source frames. This step creates the frame folders consumed by `batch_process_robust.py`:

```text
<baseline_root>/sample_<sample_id>/video_<start_time>s_png_30fps/000001.png
```

If you use a different encoding FPS, pass the same value to `get_frames.py --fps` and `batch_process_robust.py --encode_fps`.

### 4. Batch ROI encoding

This step generates Kvazaar ROI maps, converts the shared source frames to YUV, runs Kvazaar ROI encoding, and muxes the encoded stream to MP4.

Multiple bitrates are supported with one shared target resolution:

```bash
python ZocoStream/batch_process_robust.py \
  --csv_path /path/to/datasets_with_original_index.csv \
  --roi_root /path/to/roi_root \
  --baseline_root /path/to/baseline_root \
  --kvazaar_bin /path/to/kvazaar \
  --gen_script ZocoStream/gen_kvazaar_roi.py \
  --encode_fps 30 \
  --target_seconds 5 \
  --gamma 3 \
  --resolution 1920x1080 \
  --bitrate 200 400 600 910 1710 3150
```

### Notes

- You may need to change --base-url in `generate_roi.py` to match your API endpoint.
- You can **directly use** the example command above to simulate the performance in our paper.
